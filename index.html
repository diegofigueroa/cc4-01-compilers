Compiladores

Funcionamiento

Un compilador recibe un programa escrito en algún lenguaje de programación como input, y debe generar una representación de ese programa en otro lenguaje.

Este programa es tratado por el compilador como datos, un texto a procesar.

Compiladores

Análisis Léxico (Scanner)

Primera fase de un compilador.

Conocido coloquialmente como el scanner.

Su función es dividir el texto de entrada (el programa) en palabras (tokens).

Ejemplo (pizarrón):

if x == y then z = 1; else z = 2;

Compiladores

Análisis Sintáctico (Parser)

Segunda fase de un compilador.

Conocido coloquialmente como el parser.

Su función es tomar los tokens identificados por el scanner, y analizar su estructura.

En otras palabras, construir oraciones.

Árbol de sintaxis.

Ejemplo (pizarrón):

Este texto es una oración.

  • Este -> artículo
  • texto -> sustantivo
  • es -> verbo
  • una -> artículo
  • oración -> sustantivo

Análisis Semántico

Una vez analizamos la estructura podemos intentar analizar el sentido.

Examinar el sentido o significado de cada sentencia.

Esto es muy difícil para un programa, por lo que se hace de forma limitada.

Mayormente chequeos de unicidad, de tipos, control de flujo, etc.

Objetivo, encontrar incosistencias.

Tabla de símbolos.

Compiladores

IRT

Transformar el árbol de sintáxis a una estructura plana.

Utilizando instrucciones genéricas, independientes de cualquier arquitectura.

Esto facilita el trabajo del generador de código.

Compiladores

Optimización

Reemplazar instrucciones por instrucciones.

Dichos reemplazos buscan mejorar el funcionamiento del programa de acuerdo a cierto criterio:

  • Uso de memoria
  • Uso de CPU.
  • Tamaño del programa (bytes).
  • Uso de energía.
  • Etc.

Compiladores

Generación de código

Toma las instrucciones del IRT y las traduce al lenguaje objetivo.

Esto genera código ejecutable en el lenguaje objetivo.
